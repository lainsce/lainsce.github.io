Clocks, and Calendars19O05

Today, I've finished another app, called Niu. This app does one thing for me, and that is, it helps me use the Nataniev system in elementary OS. I've talked about the system, but never got to show how it works. It's composed of two concepts: Arvelie and Neralie. Both deal with timekeeping in a different way.

For Arvelie,
Each month has 2 weeks of 7 days, and each month is a letter of the alphabet. The 365th day of the year is the Year Day(+01), preceded by the Leap Day(+02) on leap years.


Jan/1 - A
Jan/2 - B
...
Dec/2 - Z

For example, now as of writing, it is 19O05. And my birthday was 93L03. (Mark that on your Arvelie calendar!)

For Neralie,
There is the concept of Beats and Pulses, and each of those custom measurements have values fixed on the second and are quite easy to understand:

1 s = 1.157 bs
1 min = 69.42 bs
1 bs = 86.4 s
1 ps = 8.64 min
1000 bs = 1 ps
1000 ps = 1 day

Neralie's notation is also quite simple:

[ps]:[bs]
example: 945:127

And here's cardinal points of interest for Neralie:
000:000 = 12 AM
250:000 = 6 AM
500:000 = 12 PM
750:000 = 6 PM
Anyhoo, hope you enjoyed learning about this system as much as I like it!

Workflow for the Day 19O04

Every day, I try to follow a strict program almost like a robot, but this programming is good for what I do, which is code at home. I always try to do at least one thing, and I have done so since 19O01 (after restarting due to backup failures). It's been a strange yet fulfilling journey. I also make sure to log it into a (big by now) text file which is structured like a Markdown table, with the task code for the day, the day task, and how many beats were used for that task.

An example:



DATE  CODE  DESC                  BEATS
19O04 1A    Logged a naxasero     7
  *   1B    Clark New Album       2
  *   2A    Write parser for this 15
  *   1A    Logged a naxasero     7

As you can see, every task has a code. To understand my codes, you need to understand that I divide things into three sections: Leisure, Work, and Research. Each of those is a number; respectively, 1, 2, and 3. A Table for each level and code follows:

Leisure (1) Work (2) Research (3)
A - Text A - Code A - Read
B - Audio B - Build B - Listen
C - Video C - Design C - Explore
D - Idle D - Idle D - Idle

That's it. So, a 1A code corresponds to a leisure activity about text. I also make sure to diversify tasks so that no task takes all the day, even if it should. The amount of beats also is a deciding factor about whether I should move on, as if a task exceeds 30 beats on the day, it's time to take a break.
